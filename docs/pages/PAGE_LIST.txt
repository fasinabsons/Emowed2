## üìù NOTES FOR DEVELOPERS

**When implementing pages:**

1. **Start with Phase 1 pages** - They're foundational
2. **Use existing components** - Reference theme.txt component library
3. **Follow database schema** - See architecture.txt
4. **Implement real-time** - Use Supabase subscriptions where specified
5. **Test mobile first** - All pages must be mobile-responsive
6. **Handle errors gracefully** - Every API call needs error handling
7. **Add loading states** - Never show blank screens
8. **Track analytics** - Add event tracking per success metrics

**Common Patterns:**
- Authentication checks in all authenticated pages
- Permission checks for role-specific pages
- Real-time subscriptions for live updates
- Optimistic UI updates for better UX
- Pagination for large lists
- Infinite scroll for feeds
- Skeleton screens for loading
- Toast notifications for actions
